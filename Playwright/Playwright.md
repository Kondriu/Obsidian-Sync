

**Глоссарий ключевых терминов:**

• **Playwright** — open-source библиотека (фреймворк) от Microsoft для автоматизации браузерных задач и сквозного (E2E) тестирования.

• **Браузерный контекст (Browser Context)** — изолированный профиль браузера, аналогичный новому окну в режиме инкогнито, позволяющий запускать тесты в полной изоляции с минимальными затратами ресурсов.

• **Авто-ожидание (Auto-wait)** — встроенный механизм, который ждет готовности элемента (видимости, кликабельности) перед выполнением действия, что устраняет необходимость в искусственных задержках.

• **Web-first assertions** — утверждения, специально разработанные для динамического веба, которые автоматически повторяют проверку до выполнения условия.

• **Трассировка (Tracing)** — инструмент записи выполнения теста, включающий видео, скриншоты, снимки DOM и логи для детального анализа ошибок.

• **Codegen** — генератор кода, записывающий действия пользователя в браузере и преобразующий их в тестовый сценарий на выбранном языке, включая Java.

• **Shadow DOM** — технология инкапсуляции в веб-компонентах; Playwright умеет «прошивать» (pierce) её автоматически при поиске селекторов.

• **Headless mode** — режим запуска браузера без графического интерфейса.

**Основные темы для изучения**

**1. Архитектура и возможности Playwright**

Playwright поддерживает все современные движки: **Chromium, WebKit и Firefox**. Одной из ключевых особенностей является работа **out-of-process** (вне процесса браузера), что избавляет от типичных ограничений внутрипроцессных инструментов. Система поддерживает мульти-акторные сценарии (несколько пользователей, вкладок и источников в одном тесте).

**2. Установка и настройка окружения**

Для Java-разработчиков Playwright доступен как библиотека. Установка возможна вручную через добавление зависимостей в проект (например, Maven/Gradle) или через команду инициализации, которая создает конфигурационные файлы и примеры. Playwright автоматически управляет установкой необходимых браузеров.

**3. Управление браузером и контекстами**

Изучение начинается с понимания иерархии: **Playwright -> Browser -> Browser Context -> Page**.

• **Изоляция:** Для каждого теста создается новый контекст, что гарантирует отсутствие влияния тестов друг на друга.

• **Авторизация:** Playwright позволяет один раз выполнить вход, сохранить состояние аутентификации и повторно использовать его во всех тестах, минуя процедуру логина.

**4. Локаторы и взаимодействие с элементами**

Playwright использует **Trusted events** (доверенные события), которые имитируют реальный ввод пользователя (клики, ввод текста, наведение курсора). Важно освоить работу с динамическими контролами, формами (Select, Checkbox, Radio) и специфическими элементами, такими как **iFrames**.

**5. Ожидания и проверки (Assertions)**

Ключ к стабильности тестов — использование встроенных механизмов:

• **Авто-ожидания** перед действиями.

• **Explicit waits** (явные ожидания) для специфических условий.

• **Assertions**, которые автоматически повторяются (retries) до успеха или таймаута.

**6. Работа с сетью и API**

Playwright предоставляет мощные возможности для перехвата и **мокирования API-запросов**. Это позволяет тестировать фронтенд в изоляции или комбинировать UI и API проверки в рамках одного сценария для повышения скорости тестирования.

**7. Отладка и инструментарий**

Эффективная разработка тестов невозможна без:

• **Playwright Inspector** — пошаговое выполнение и генерация селекторов.

• **Trace Viewer** — визуальный анализ проваленных тестов.

• **Снятия скриншотов и записи видео**.

**8. Масштабирование и CI/CD**

Для профессиональной работы необходимо освоить:

• **Параллельный запуск** тестов (например, через JUnit).

• **Параметризацию** тестов.

• **Интеграцию с CI/CD** (GitHub Actions, Azure Pipelines).

• Облачное тестирование через **Microsoft Playwright Testing**, позволяющее запускать до 50 тестов параллельно в облаке.

**Практические примеры**

В источниках упоминаются следующие кейсы для реализации:

• **Базовая навигация:** Перейти на страницу, сделать скриншот и сохранить его в файл.

• **Эмуляция мобильных устройств:** Запуск тестов с параметрами Mobile Safari, настройка геолокации и проверка отображения карт.

• **Выполнение скриптов:** Выполнение произвольного JavaScript-кода непосредственно в контексте страницы.

• **Логирование сети:** Перехват всех сетевых запросов страницы и вывод их в консоль.

• **Тестирование Title:** Переход на сайт и проверка заголовка страницы с помощью утверждения `toHaveTitle`.
